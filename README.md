# PC-ERP Pro - Firebase 통합 재고관리 시스템

## 🎯 주요 기능

### 1. 재고 현황 관리
- 품목별 재고 조회 (본사 창고 / 매장 재고)
- 재고 등록, 수정, 삭제
- 안전 재고 미달 알림
- 실시간 재고 동기화

### 2. 입출고 관리
- **입고**: 본사 또는 매장으로 재고 입고
- **출고**: 본사 또는 매장에서 재고 출고
- **이동**: 본사 ↔ 매장 간 재고 이동
- **자동 재고 업데이트**: 거래 등록 시 재고 현황 자동 반영

### 3. 거래처 정보 관리
- 거래처 등록, 수정, 삭제
- 여신 한도 관리
- 거래처별 상세 정보 관리

### 4. 시리얼 번호 추적
- 제품별 시리얼 번호 관리
- 시리얼 번호 검색 및 추적

## 🔥 실시간 재고 업데이트 동작 방식

### 입출고 거래 등록 시 자동 재고 반영

```
1. 사용자가 "입출고 관리" 탭에서 거래 등록
   ↓
2. 거래 내역이 Firestore에 저장됨
   ↓
3. 동시에 해당 품목의 재고 수량이 자동 업데이트
   ↓
4. Firebase 실시간 리스너가 변경사항 감지
   ↓
5. "재고 현황" 탭이 자동으로 새로고침됨 (페이지 새로고침 불필요!)
```

### 예시

**입고 거래 등록:**
- 품목: CPU-001 (Intel i7)
- 수량: 10개
- 창고: 본사
- 결과: 본사 창고 재고 +10개 자동 증가

**출고 거래 등록:**
- 품목: CPU-001 (Intel i7)
- 수량: 5개
- 창고: 본사
- 결과: 본사 창고 재고 -5개 자동 감소

**이동 거래 등록:**
- 품목: CPU-001 (Intel i7)
- 수량: 3개
- 창고: 본사 → 매장
- 결과: 본사 -3개, 매장 +3개 자동 반영

## 🚀 시작하기

### 1. Firebase 설정
`FIREBASE_SETUP.md` 파일을 참고하여 Firebase 프로젝트를 설정하세요.

### 2. Firestore 보안 규칙 설정
`FIRESTORE_RULES.md` 파일의 규칙을 Firebase Console에 적용하세요.

### 3. 개발 서버 실행
```bash
npm install
npm run dev
```

### 4. 계정 생성 및 로그인
1. http://localhost:3000 접속
2. "시작하기" 버튼 클릭
3. 회원가입 또는 로그인

## 📝 사용 방법

### 재고 등록
1. "재고 현황" 탭 선택
2. "신규 등록" 버튼 클릭
3. 품목 정보 입력 (품목코드, 품목명, 카테고리, 수량 등)
4. "등록하기" 버튼 클릭

### 입출고 거래 등록
1. "입출고 관리" 탭 선택
2. "거래 등록" 버튼 클릭
3. 거래 정보 입력:
   - 거래 구분 선택 (입고/출고/이동)
   - 품목 선택
   - 수량 입력
   - 창고 선택
4. "등록하기" 버튼 클릭
5. **재고 현황 탭으로 이동하면 자동으로 반영된 재고 확인 가능!**

### 거래처 등록
1. "거래처 정보" 탭 선택
2. "거래처 등록" 버튼 클릭
3. 거래처 정보 입력
4. "등록하기" 버튼 클릭

## 📊 데이터 구조

### Firestore Collections

- **inventory**: 재고 정보
  - `id`: 품목코드
  - `name`: 품목명
  - `main`: 본사 창고 수량
  - `shop`: 매장 재고 수량
  - `total`: 총 재고 (자동 계산)
  - `userId`: 사용자 ID

- **transactions**: 입출고 거래 내역
  - `type`: 거래 구분 (입고/출고/이동)
  - `productId`: 품목코드
  - `quantity`: 수량
  - `warehouse`: 창고
  - `userId`: 사용자 ID

- **partners**: 거래처 정보
  - `id`: 거래처코드
  - `name`: 상호
  - `creditLimit`: 여신 한도
  - `userId`: 사용자 ID

- **serials**: 시리얼 번호
  - `serialNumber`: 시리얼 번호
  - `productId`: 품목코드
  - `userId`: 사용자 ID
